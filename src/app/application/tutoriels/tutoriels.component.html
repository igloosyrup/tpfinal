C'est quoi Angular?

Angular est un frameworkpour développement web gratuit (Open Source) SPA (Single Page Application) en Typescript. Il permet au développeurs de coder plus rapidement grâce aux commandes angulars qui permet de générer des templates.

Angular est un framework Open Source pour le developpement front-end rapide de pages web SPA à l'aide du Typescript.

Un site dévelopé avec Angular est adapté pour toute platforme (Ordinateur, Mobile, etc.) et Responsive

De plus, Angular est un des frameworks Javascript les plus populaire principalement maintenu par google.

mini tutoriel c.r.u.d.

C'est quoi le C.R.U.D.?

Le terme CRUD est composé de 4 termes. Il s'agit de:

  • Create
  • Read
  • Update
  • Delete

    • En effet, ce sont les 4 opérations qui permet de manipuler des données stockées ou non.

C'est quoi Create?

Le mot create en anglais veut dire Créer en français, et cela signifie en informatique l'insertion d'une nouvelle entrée (dans un contexte de base de données), la création d'une nouvelle instance d'un objet (pour la programmation objet orienté) ou une requête HTTP POST.

C'est quoi Read?

Le terme read en anglais veut dire Lire en français. Le Read est essentiellement une méthode ou opération qui permet de lire/afficher les données que ce soit pour une base de données d'objets (en programmation objet orienté) ou d'une requête HTTP GET, getAll(), getByID(), etc.

C'est quoi Update?

Le terme update en anglais veut dire Mettre-à-jour en français. Le update permet de mettre-à-jour des données et cela se fait à l'aide d'une requête UPDATE (dans un context de base de données), d'une requête PUT (dans un contexte HTTP) et finallement créer une méthode/fonction pour mettre-à-jour les données (Objets et non-objets).

C'est quoi Delete?

Delete signifie la suppression. La suppression de données est souvent appelé comme suit: la requête DELETE (en base de données et en HTTP)
tandis qu'en programmation orienté objet, c'est à l'utilisateur de déterminer.

mini tutoriel relations

C'est quoi les relations?

Les relations permet determiner comment les objets(programmation objet orienté)/tables(context base de données) vont s'intéragir entre-eux et comment ces-derniers seront stockées dans une base de données. En effet, les relations sont nécessaires lorsqu'il y a un besoin de stocker des données dans une base de données ou le besoin de programmer objet orienté.

Il existe 6 relations possibles.

Les 6 flèches de relations avec exemples:

L'association : Une classe A (un(e) action/verbe) mange une classe B ou B mange A.
Multiplicité possible: One to One, One To Many (vice versa) ou Many To Many
Nombre de relations avec différentes classes possibles: Plusieurs
Direction de la flèche: Uni ou Bidirectionnel

L'héritage/généralisation : Une classe B hérite (extends en Java, Typescript) de la classe A.
Multiplicité possible: One to One, One To Many (vice versa) ou Many To Many
Nombre de relations avec autres classes possibles: Un seul
Direction de la flèche: Unidirectionel

La dépendance : Classe A dépend de Classe B. (lorsqu'il y a modifcation dans la classe B, la classe A les aura aussi)
Multiplicité possible: One to One et Many to One
Nombre de relations avec autres classes possibles: Un seul
Direction de la flèche: Unidirectionel

Composition : Classe A est composé de Classe B, une nouvelle instance A doit absolument créer une classe B dans son constructeur.
IMPORTANT la suppréssion est bidirectionel, C'est-à-dire que les classes liées à la composition sont supprimées
Voici un exemple d'une composition:
Class A { b:B;
constructor(){ this.b = new B() }
}
Multiplicité possible: One to One, One To Many (vice versa) ou Many To Many
Nombre de relations avec autres classes possibles: Plusieurs (Recommendé* un Seul)
Direction de la flèche: Unidirectionel

Aggrégation : Classe A est en Aggrégation avec Classe B. Contrairement à la composition, lorsque une classe est supprimé l'autre classe ne l'ait pas.
Voici un exemple d'une aggrégation:
Class A { b:B;
constructor(b:B){ this.b = b }
}
Multiplicité possible: One to One, One To Many (vice versa) ou Many To Many
Nombre de relations avec autres classes possibles: Plusieurs
Direction de la flèche: Unidirectionel

La multiplicité

Il existe trois types possibles:

One-To-One : Classe A 0,1 ... 0,1 B Classe B

One-To-Many : Classe A 0,1 ... 0,n Classe B ou Classe A 0,n ... 0,1 Classe B

Many-To-Many : Classe A 0,n ... 0,n Classe B
*Remarque: lorsqu'il y a un many to many, la relation many to many nécessite une troisième table qui contiendra la relation entre Classe A et Classe B
